# Game Factory Platform — Production Audit

This document describes the extended "mobile game factory" platform for generating, launching, and analyzing many mobile games.

---

## 1. Architecture overview

The repository is a **pnpm monorepo** with:

- **apps/** — One directory per game (Expo/React Native). Each has `src/` (game logic, config, metadata, features, theme, i18n), `metadata/` (app store copy, ASO, screenshots), and `assets/`.
- **packages/** — Shared libraries: `app-shell`, `game-core`, `storage`, `analytics`, `monetization`, and **analytics-dashboard** (Node server + static UI to compare games).
- **scripts/** — Generator and tooling:
  - `create-game.js` — Scaffold one game from template (metadata, features, difficulty, genre, store metadata).
  - `create-games.js` — Batch: run create-game for multiple names; failures do not stop the batch.
  - `create-game-assets.js` — Generate icon, splash, favicon (sharp + SVG).
  - `generate-aso-keywords.js` — Write `metadata/aso-keywords.json` from game name + genre.
  - `generate-screenshots.js` — Placeholder screenshots (iPhone, Android, tablet) with title, mock gameplay, score.
  - `generate-test-data.js` — Fake scores, events, ad interactions for QA and dashboard.
- **docs/** — Architecture and audits (this file, game-factory-audit, create-game audits).

Data flow: Games emit events via `@bitcraft/analytics`. The dashboard reads JSON under `packages/analytics-dashboard/data/` (e.g. generated by `generate-test-data.js`) to show sessions, high scores, playtime, ad interactions per game.

---

## 2. Generator capabilities

**create-game** now:

- Copies `apps/template` and replaces all template identifiers (word boundaries).
- Writes **gameConfig.ts** (gameId, maxLives, difficulty with mode/speedMultiplier/spawnRate/dynamicDifficulty).
- Writes **gameMetadata.ts** (gameId, title en/es/de, shortDescription, keywords, category, subcategory).
- Writes **features.ts** (ads, analytics, inAppPurchases, haptics, sound) from flags: `--minimal` → analytics/ads false; `--with-ads` → ads true.
- Creates **metadata/** with **appstore.json** and **playstore.json** (title, subtitle, description, keywords, supportUrl, privacyUrl) derived from gameMetadata.
- Optionally generates **assets/** (icon, adaptive-icon, splash, favicon) unless `--no-assets`.
- Adds root scripts: `dev:<game>`, `ios:<game>`, `android:<game>`, `web:<game>`.
- **--genre arcade|puzzle|runner** sets ASO keywords, metadata subcategory, and difficulty defaults.
- Runs validation (install, typecheck, expo config); on failure removes the generated app.

**create-games** runs create-game for each argument; one failure does not stop the rest.

---

## 3. Metadata system

- **gameMetadata.ts** (per app): `gameId`, `title` (en, es, de), `shortDescription` (en, es), `keywords` array, `category` ("games"), `subcategory` (e.g. "arcade", "puzzle", "runner"). Used for store listings and ASO.
- **metadata/appstore.json**: title, subtitle, description, keywords, supportUrl, privacyUrl (App Store).
- **metadata/playstore.json**: title, shortDescription, fullDescription, keywords array, supportUrl, privacyUrl (Play Store).

Values are derived from game name, display name, and genre. Edit these files before submitting to stores.

---

## 4. ASO system

- **generate-aso-keywords.js** — Input: game name, optional genre. Output: **apps/&lt;game-name&gt;/metadata/aso-keywords.json** with a top-keywords list by genre (e.g. arcade → arcade, casual, retro, fast, highscore, endless, action).
- Genre mapping: arcade, puzzle, runner each have a fixed keyword set; game name is appended.
- Usage: after create-game, run `pnpm generate-aso-keywords &lt;game-name&gt; [genre]` to refresh keywords.

---

## 5. Screenshot generation

- **generate-screenshots.js** — Generates placeholder screenshots under **apps/&lt;game-name&gt;/metadata/screenshots/**.
- Sizes: iPhone 6.5", iPhone 5.5", Android phone, tablet 12".
- Each image: dark background, “mock gameplay” area, game title, score overlay. Implemented with sharp (SVG → PNG).
- Usage: `pnpm generate-screenshots &lt;game-name&gt;`. Replace with real screenshots before store submission.

---

## 6. Analytics dashboard

- **Package:** `@bitcraft/analytics-dashboard`.
- **Purpose:** Compare performance across games (list games, daily sessions, high scores, average playtime, ad interactions).
- **Data source:** JSON files in **packages/analytics-dashboard/data/** (e.g. `&lt;game-name&gt;-test-data.json` produced by **generate-test-data.js**). Events from `@bitcraft/analytics` can be persisted and placed here for production.
- **Run:** `pnpm dashboard` (starts Node server on port 3999).
- **Endpoints:** `/api/games` — aggregated per game (sessions, total playtime, high score, ad interactions). `/api/data` — raw datasets. `/` — simple HTML table of games.
- **Workflow:** Run `generate-test-data.js` for one or more games, then open the dashboard to inspect metrics.

---

## 7. Feature flags

- **features.ts** (per app): `ads`, `analytics`, `inAppPurchases`, `haptics`, `sound`.
- Generator rules:
  - **Default (full):** all true.
  - **--minimal:** analytics false, ads false, inAppPurchases false.
  - **--with-ads:** ads true even with --minimal.
  - **--with-analytics:** analytics true even with --minimal.

Use these flags in app code to gate analytics calls, ad placements, and IAP.

---

## 8. Batch generation

- **Script:** `scripts/create-games.js`.
- **Usage:** `pnpm create-games game1 game2 game3 [--minimal] [--no-assets] [--genre arcade] ...`
- **Behavior:** For each game name, runs `create-game` with the same flags. If one fails, the script logs and continues; exit code 1 if any failed.
- **Use case:** Launch many games in one go; fix failures individually and re-run for missing names.

---

## 9. Theme-aware assets

- **Default:** Generator produces **assets/icon.png**, **assets/adaptive-icon.png**, **assets/splash.png**, **assets/favicon.png** (single set).
- **Optional:** If you add **assets/light/** and **assets/dark/** (e.g. light/icon.png, dark/icon.png), the app can choose theme-specific icons at runtime (e.g. from theme preference). The generator does not create these by default; fallback remains the default assets in **assets/**.

---

## 10. QA tools

- **generate-test-data.js** — Generates fake data for a game: fake scores, analytics events (app_opened, game_over), ad interactions (placements, impression/click). Output: **packages/analytics-dashboard/data/&lt;game-name&gt;-test-data.json**.
- **Usage:** `node scripts/generate-test-data.js &lt;game-name&gt; [days=7]`. Use for testing the analytics dashboard and any UI that displays sessions, high scores, or ad metrics.

## Example workflows for launching multiple games

**1. Create three arcade games and open dashboard**

```bash
pnpm create-games flappy runner puzzle-game --genre arcade
pnpm install
node scripts/generate-test-data.js flappy 14
node scripts/generate-test-data.js runner 14
node scripts/generate-test-data.js puzzle-game 14
pnpm dashboard
# Open http://localhost:3999
```

**2. Create one game with full metadata and store assets**

```bash
pnpm create-game mygame --genre runner
pnpm generate-aso-keywords mygame runner
pnpm generate-screenshots mygame
# Edit apps/mygame/metadata/appstore.json and playstore.json, replace screenshots with real ones
```

**3. Minimal batch for quick experiments**

```bash
pnpm create-games proto1 proto2 proto3 --minimal --no-assets
```

**4. Single game with ads and puzzle genre**

```bash
pnpm create-game block-puzzle --genre puzzle --with-ads
pnpm generate-aso-keywords block-puzzle puzzle
```
